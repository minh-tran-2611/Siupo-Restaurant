# 🌐 Internationalization (i18n)

## Overview

The application supports **English** and **Vietnamese** using **i18next** framework.

---

## 🏗️ Architecture

### Technology Stack

- **i18next** (25.6.3) - Core i18n framework
- **react-i18next** (16.3.5) - React bindings
- **i18next-browser-languagedetector** (8.2.0) - Auto language detection

### Namespace Structure

12 organized translation namespaces:

| Namespace | Purpose | Example Keys |
|-----------|---------|--------------|
| `common` | Global UI, navigation, actions | `navigation.home`, `actions.submit` |
| `home` | Home page content | `hero.title`, `stats.customers` |
| `auth` | Authentication pages | `signIn.title`, `signUp.email` |
| `checkout` | Checkout process | `address.fullName`, `payment.method` |
| `cart` | Shopping cart | `item.product`, `summary.total` |
| `product` | Product pages | `detail.description`, `reviews.rating` |
| `booking` | Table reservations | `form.date`, `success.message` |
| `account` | User account pages | `dashboard.title`, `settings.profile` |
| `about` | About us page | `story.title`, `mission.description` |
| `menu` | Menu page | `categories.all`, `filter.price` |
| `contact` | Contact page | `form.name`, `info.phone` |
| `chef` | Chef profiles | `card.specialty`, `profile.bio` |

## ⚙️ Configuration

### i18n Setup

**File:** `src/i18n/config.ts`

```typescript
import i18n from "i18next";
import LanguageDetector from "i18next-browser-languagedetector";
import { initReactI18next } from "react-i18next";

// Import translations
import commonEn from "../locales/en/common.json";
import commonVi from "../locales/vi/common.json";
// ... import other namespaces

export const resources = {
  en: {
    common: commonEn,
    home: homeEn,
    auth: authEn,
    // ... other namespaces
  },
  vi: {
    common: commonVi,
    home: homeVi,
    auth: authVi,
    // ... other namespaces
  },
} as const;

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources,
    fallbackLng: "vi",              // Default language
    defaultNS: "common",             // Default namespace
    ns: [
      "common", "home", "auth", 
      "checkout", "cart", "product",
      // ... all namespaces
    ],
    detection: {
      order: ["localStorage", "navigator"],
      lookupLocalStorage: "i18nextLng",
      caches: ["localStorage"],
    },
    interpolation: {
      escapeValue: false,            // React escapes by default
    },
  });

export default i18n;
```

### TypeScript Definitions

**File:** `src/i18n/i18next.d.ts`

```typescript
import "i18next";
import { resources } from "./config";

declare module "i18next" {
  interface CustomTypeOptions {
    resources: typeof resources["en"];
  }
}
```

### Custom Hook

**File:** `src/hooks/useTranslation.ts`

```typescript
import { useTranslation as useI18nTranslation } from "react-i18next";

export type Namespace =
  | "common"
  | "home"
  | "auth"
  | "checkout"
  | "cart"
  | "product"
  | "booking"
  | "account"
  | "about"
  | "menu"
  | "contact"
  | "chef";

export function useTranslation(namespace: Namespace = "common") {
  return useI18nTranslation(namespace);
}

export default useTranslation;
```

## 💻 Usage Examples

### Basic Usage (Common Namespace)

```tsx
import { useTranslation } from '../../hooks/useTranslation';

function MyComponent() {
  // Default to 'common' namespace
  const { t } = useTranslation();
  
  return (
    <div>
      <h1>{t('navigation.home')}</h1>
      <button>{t('actions.submit')}</button>
      <p>{t('messages.success')}</p>
    </div>
  );
}
```

### Specific Namespace

```tsx
import { useTranslation } from '../../hooks/useTranslation';

function CheckoutPage() {
  const { t } = useTranslation('checkout');
  
  return (
    <div>
      <h1>{t('title')}</h1>
      <form>
        <input placeholder={t('address.fullName')} />
        <input placeholder={t('address.phone')} />
        <input placeholder={t('address.street')} />
      </form>
    </div>
  );
}
```

### Multiple Namespaces

```tsx
import { useTranslation } from '../../hooks/useTranslation';

function SignUpPage() {
  const { t } = useTranslation('auth');
  const { t: tCommon } = useTranslation();
  
  return (
    <div>
      <h1>{t('signUp.title')}</h1>
      <button>{tCommon('actions.submit')}</button>
    </div>
  );
}
```

### With Interpolation

```tsx
// Translation file
{
  "welcome": "Welcome, {{name}}!",
  "itemsCount": "You have {{count}} items"
}

// Component
const { t } = useTranslation();

<p>{t('welcome', { name: user.name })}</p>
<p>{t('itemsCount', { count: 5 })}</p>
```

### With Pluralization

```tsx
// Translation file
{
  "items": "{{count}} item",
  "items_other": "{{count}} items"
}

// Component
<p>{t('items', { count: 1 })}</p>  // "1 item"
<p>{t('items', { count: 5 })}</p>  // "5 items"
```

### Change Language

```tsx
import { useTranslation } from '../../hooks/useTranslation';

function LanguageSwitcher() {
  const { i18n } = useTranslation();
  
  const changeLanguage = (lang: string) => {
    i18n.changeLanguage(lang);
  };
  
  return (
    <div>
      <button onClick={() => changeLanguage('en')}>
        English
      </button>
      <button onClick={() => changeLanguage('vi')}>
        Tiếng Việt
      </button>
    </div>
  );
}
```

### Get Current Language

```tsx
const { i18n } = useTranslation();
const currentLang = i18n.language; // 'en' or 'vi'

// Check if specific language
if (i18n.language === 'vi') {
  // Vietnamese-specific logic
}
```

## 🎯 Best Practices

### 1. Consistent Key Naming

Use dot notation for nested keys:

```json
{
  "section": {
    "subsection": {
      "key": "value"
    }
  }
}
```

Access: `t('section.subsection.key')`

### 2. Meaningful Key Names

✅ **Good:**
```json
{
  "form": {
    "email": "Email Address",
    "emailPlaceholder": "Enter your email",
    "emailError": "Invalid email"
  }
}
```

❌ **Bad:**
```json
{
  "field1": "Email Address",
  "placeholder1": "Enter your email",
  "error1": "Invalid email"
}
```

### 3. Group Related Keys

```json
{
  "product": {
    "detail": {
      "name": "Product Name",
      "price": "Price",
      "description": "Description"
    },
    "actions": {
      "addToCart": "Add to Cart",
      "addToWishlist": "Add to Wishlist"
    }
  }
}
```

### 4. Use Namespaces Appropriately

- **Common namespace:** Global UI elements used everywhere
- **Page namespace:** Page-specific content

```tsx
// ✅ Use common for global actions
const { t } = useTranslation();
<button>{t('actions.submit')}</button>

// ✅ Use page namespace for page content
const { t } = useTranslation('checkout');
<h1>{t('title')}</h1>
```

### 5. Keep Translations Consistent

Ensure same meaning across languages:

```json
// en/common.json
{ "actions": { "submit": "Submit" } }

// vi/common.json
{ "actions": { "submit": "Gửi" } }
```

### 6. Handle Missing Translations

```tsx
// Fallback to English if translation missing
const { t } = useTranslation();

// Returns key if translation not found
t('missing.key') // → "missing.key"
```

## 🔧 Advanced Features

### Context-Specific Translations

```json
{
  "cart": {
    "empty_male": "Your cart is empty, sir",
    "empty_female": "Your cart is empty, madam"
  }
}

// Usage
t('cart.empty', { context: user.gender })
```

### Default Values

```tsx
// If key doesn't exist, use default
t('missing.key', { defaultValue: 'Default text' })
```

### HTML in Translations

```tsx
// Translation
{ "terms": "I agree to the <1>Terms of Service</1>" }

// Component
<Trans i18nKey="terms">
  I agree to the <a href="/terms">Terms of Service</a>
</Trans>
```

## 🐛 Troubleshooting

### Translation Keys Not Showing

**Problem:** Keys display instead of translations

**Solutions:**
1. Check key exists in JSON file
2. Verify namespace is correct
3. Ensure file is imported in `i18n/config.ts`
4. Restart dev server

### Language Not Changing

**Problem:** Language switcher doesn't work

**Solutions:**
1. Check localStorage for `i18nextLng` key
2. Clear browser cache
3. Verify i18n.changeLanguage() is called
4. Check console for errors

### TypeScript Errors

**Problem:** Type errors with translation keys

**Solutions:**
1. Check `i18next.d.ts` file exists
2. Restart TypeScript server in VS Code
3. Ensure resources are properly typed
